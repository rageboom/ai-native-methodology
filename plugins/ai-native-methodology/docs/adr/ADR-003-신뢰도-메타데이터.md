# ADR-003: 신뢰도 메타데이터 표준

- 상태: 승인됨 (Accepted)
- 일자: 2026-04-26 (v1.1.1에서 §6~§10 확장)
- 결정자: 윤주스 (TF Lead)
- 관련: ADR-001, ADR-002, schemas/meta-confidence.schema.json

## 컨텍스트

AI가 추출한 산출물은 100% 정확하지 않다. 모든 산출물에 **"이 결과를 얼마나 신뢰할 수 있는가?"**를 수치화해야 한다.

## 결정

**모든 산출물에 신뢰도 메타데이터를 필수로 포함**한다.

### 메타데이터 구조

```yaml
meta:
  generated_at: 2026-04-26T10:00:00Z
  source_commit_sha: abc1234
  formula_version: "v1"
  inputs_used: [source_code, erd, orm, operational_db, planning_docs, design_specs, domain_context_md, postman_or_api_test, diagrams_other]
  confidence: 0.85       # 0.0~0.98 (cap 적용)
  confidence_breakdown:
    - area: "도메인 모델"
      confidence: 0.90
      element_count: 12
      extraction_method: pattern_matching
  applied_modifiers: []
  applied_penalties: []
  cap_applied: false
  manual_override: false
  human_review_required:
    - "BR-ORDER-007: 19세 기준의 의도 확인 필요"
```

### §6: 산정 공식 v1 (가법 + cap 0.98)

```
confidence = min(0.98, base + Σ(modifiers) - Σ(penalties))
base_confidence = 0.75 (소스만)
```

**가산점**: ERD(DBML) +0.10, ORM(full) +0.10, 운영DB +0.08, 기획문서(구조화) +0.05, 디자인명세 +0.05, Postman +0.05, 도메인컨텍스트 +0.03, ERD(이미지) +0.03, 기획(비구조화) +0.02

**페널티**: 출처간 불일치 -0.05, ORM 없음 -0.05, 문서화 부재 -0.03, generated 코드 포함 -0.03, 매직넘버 다수 -0.02

### §7: 영역별 가중 평균 (요소 수 가중)

```
weighted_avg = Σ(area.confidence × area.element_count) / Σ(area.element_count)
total = min(0.98, weighted_avg)
```

### §8: 추출 방법별 신뢰도

| 방법 | 범위 | 예시 |
|---|---|---|
| 결정적 | 0.95~0.98 | AST 파싱, DB 메타 |
| 패턴 매칭 | 0.80~0.90 | ORM @Entity 감지 |
| LLM + grounding | 0.50~0.75 | domain-context 참조 추론 |
| LLM (코드만) | 0.40~0.60 | 코드만 보고 의도 추론 |

### §9: 해석 가이드

| 범위 | 해석 | 액션 |
|---|---|---|
| ≥0.95 | 거의 확실 | 검토 생략 가능 |
| 0.80~0.95 | 신뢰 가능 | 샘플 검토 권장 |
| 0.60~0.80 | 참고 수준 | 전수 검토 권장 |
| 0.50~0.60 | 낮음 | 전수 검토 필수 |
| <0.50 | 매우 낮음 | 차단 |

> ⚠️ confidence ≠ accuracy. 신뢰도는 추정치이지 정답률이 아니다.

### §10: v1 한계

- 가산점/페널티 값은 경험적 추정 (calibration 데이터 부재)
- v1.2에서 PoC 4~5건 누적 후 calibration 예정
- v2 후보: 베이지안 모델

## 결정 근거

research-f003-신뢰도공식.md에서 가법/곱셈/베이지안 등 6가지 비교 후 **가법+cap** 채택. 단순하고 해석 쉽고 PoC 검증 가능.
