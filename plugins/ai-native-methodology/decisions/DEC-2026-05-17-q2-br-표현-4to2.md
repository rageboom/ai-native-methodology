## Lessons Learned 등재 ( session 26차 / ADR-CHAIN-011 §9)

- **LL-i-54** — "표현 자격 축소(②)는 alias 폐기(①)와 schema 메커니즘 동형(allOf anyOf 축소)이나 PoC 데이터 비용은 ①≪② 비대칭 (① = 순수 기계 rename / ② = 도메인 semantic 합성 #06 7 BR). breaking 동형이라고 비용 동형 가정 ❌ — STOP-1 dry-run 으로 합성 총량·회귀면 사전 정량 의무."
- **LL-i-55** — "schema branch 제거 시 해당 필드의 외부 consumer(load-bearing) 실측 의무 — `decision-table-validator` 가 expected_result/verification_location/rejection_method 를 REQUIRED 로 소비 = naive hard-remove(D2 (A)) 가 silent breakage. official-docs#2(branch 제거 ≠ property 금지) + Senior close round 실소스 검증으로 'branch only 제거 + property 보존' 정답 도출. 가정 2개를 코드 착수 전 실소스로 닫는 close round paradigm (confidence 0.82→0.90)."
