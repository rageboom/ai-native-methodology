## 사용자 결단 (2026-05-06)

**(A) i-strict 채택** — 본 방법론 = SDLC 4단계 chain harness:

```
INPUT (1차 = legacy):
  ↓ analysis stage (현 v1.5.0 자산)
  ↓
[CHAIN 1] planning-spec  ── go/stop gate #1
  ↓
[CHAIN 2] behavior-spec + acceptance-criteria + 7대 산출물 통합  ── go/stop gate #2
  ↓
[CHAIN 3] test-spec + 실 test 코드 (RED)  ── go/stop gate #3
  ↓
[CHAIN 4] impl-spec + 실 impl 코드 (GREEN / 100% test pass)  ── go/stop gate #4
  ↓
OUTPUT: prod 시스템 + traceability-matrix
```

revisit loop: 도구 자동 감지 → 사용자 prompt → go/stop. stop 결단 시 임의 stage jump.
