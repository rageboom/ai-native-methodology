## 5. Lessons Learned

1. ** release note 의 carry 환경 가정 (Docker only) 이 채널 다양성 검증 누락 패턴** — pip / pipx / uv 등 다른 진짜 도구 채널 옵션 검증 의무. carry 명시 시 "환경 가정" 컬럼 별도 + "다른 채널 가능성 검증 여부" 명시 의무 ( memory `feedback_carry_environment_assumption.md` 신규).

2. ** 진짜 도구 첫 실행 = 본체 도구 bug 발견 패턴 입증** — no-simulation 정책 첫 적용 시 본체 도구 자체의 bug (result_hash null / source_commit_sha unknown) 발견 가능. 즉 no-simulation 정책 = 도구 자체의 가치 검증 효과까지 동반. feedback_no_simulation_realized.md 패턴 강화.

3. ** Senior research 추정 over-estimate 입증** — RealWorld FSD 코드 품질 우수 (66 files / 76 rules / 0 findings). Senior 가 추정한 20~75 finding / FP 40~60% / 분류 2~6시간 = 모두 over-estimate. ** 의외로 좋은 결과** — Phase E 시간 30분 → 10분 단축 (분류 부담 0).

4. ** Silver bullet 부재 입증** — Semgrep p/owasp-top-ten (커뮤니티 표준 ruleset) 만으로 4 PoC isomorphic AP-FE-SECURITY-JWT-LOCALSTORAGE 매칭 불가. 사내 적용 시 custom rule 작성 의무 ( adoption 트랙 또는 v1.4.2 PATCH).
