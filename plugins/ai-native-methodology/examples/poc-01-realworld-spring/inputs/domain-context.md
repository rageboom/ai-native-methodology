# 도메인 컨텍스트 — RealWorld (Conduit)

> 본 문서는 Phase 4 LLM 호출 시 **grounding** 으로 사용된다.
> Phase 0 산출 (사용자 작성 권장) — TCS Research domain context grounding 패턴.

---

## 비즈니스 영역

이 시스템은 **소셜 블로깅 플랫폼** 도메인을 다루는 **Medium 클론** 시스템이다 (RealWorld spec, 코드명 "Conduit"). 사용자가 글을 쓰고 다른 사람의 글을 읽고, 팔로우하고, 좋아요/태그를 통해 발견하는 모델.

학습/데모용 표준 spec — 의도적으로 단순한 도메인.

---

## 주요 사용자

- **일반 사용자**: 글 작성/읽기/댓글/좋아요/팔로우
- **비로그인 방문자**: 글 읽기, 태그 탐색
- **(없음) 관리자**: RealWorld spec에는 관리자 역할 없음

⚠️ **참고**: 한국 엔터프라이즈와 다른 점 — 결제/PG 없음, SMS/Email 없음, 외부 통합 없음.

---

## 핵심 비즈니스 흐름

### 흐름 1: 회원가입 → 첫 글 작성
1. 회원가입 (email, username, password)
2. 로그인 (JWT 발급)
3. 글 작성 (title, description, body, tagList)
4. 발행 → 글 목록에 표시

### 흐름 2: 글 발견 → 팔로우/좋아요
1. 글 목록 조회 (페이지네이션)
2. 태그로 필터
3. 글 상세 조회
4. 좋아요 (favorite)
5. 작성자 프로필 → 팔로우

### 흐름 3: 댓글 + 토론
1. 글 상세 진입
2. 댓글 작성
3. 댓글 삭제 (작성자만)
   ⚠️ 댓글 수정은 RealWorld spec에 없음

### 흐름 4: 개인화된 피드
1. 팔로우한 사용자의 글만 표시
2. (Global) 모든 글 표시 vs (Your Feed) 팔로우만

## 알려진 제약/규칙 (RealWorld spec)

### 인증/권한
- JWT 기반 인증 (Bearer token)
- 회원가입/로그인은 익명
- 글 수정/삭제는 작성자만
- 댓글 삭제는 작성자만 (수정 자체가 없음)
- 팔로우/언팔로우는 본인이 본인을 못 함

### 데이터 제약
- email, username은 unique
- Article slug는 unique (자동 생성)
- 댓글은 수정 불가 (CR D만)

### 비즈니스 규칙 (예상)
- (RealWorld spec은 비즈니스 정책 거의 없음 — 단순 CRUD 위주)
- 의외로 발견될 수 있는 정책 후보:
  - 글 길이 제한? (검증 필요)
  - 태그 개수 제한?
  - 댓글 길이 제한?
  - 팔로우 수 제한?

→ Phase 4 5.A/5.B에서 발견되면 BR로 등록.

## 한국 엔터프라이즈와의 차이 (이 PoC의 한계)

⚠️ 이 도메인은 **단순한 학습용** spec이다. 사내 진짜 적용 시 차이:

| 영역 | RealWorld | 한국 엔터프라이즈 |
|---|---|---|
| 외부 통합 | 없음 | PG/SMS/SSO 다수 |
| 비즈니스 규칙 | 거의 없음 | 정책 100+ 흔함 |
| 환경별 설정 | 단순 | 환경별 정책 차이 흔함 |
| 권한 모델 | 단순 (작성자/타인) | RBAC, 다단계 권한 |
| 다국어 | 영어만 | 한/영 동시 |
| 결제 | 없음 | 핵심 |

→ 본 PoC가 80% 영역 검증, 나머지 20%(특히 5.D, 복합 안티패턴)는 사내 PoC에서.
