
## Why

비교 기준 (Stage 5 본격 분석 cross-reference):

- **Login** (login.ui.tsx) = email + password 모두 `required` ✅
- **Register** (register.ui.tsx) = username + email + password 모두 `required` ✅
- **Settings** (settings.ui.tsx) = username + email `required` ✅ / password optional (✅ update)
- **CommentCreate** (article.ui.tsx) = body `required` ✅
- **Editor** (editor.ui.tsx) = 4 input 모두 `required` ❌

→ **Editor 만 일관성 위반** — 다른 5 form 모두 HTML5 native required 적용. Zod action 단계 검증은 ✅ (validateSchema NewArticleRequest / UpdateArticleRequest) 단 클라이언트 1차 UX 검증 누락 (submit 후 server response 까지 대기).

## Severity

medium — UX 문제 (1차 검증 누락 / submit 시점에야 error 표시).

## Recommendation

```diff
- <input type="text" name="title" placeholder="Article Title" defaultValue={title} />
+ <input type="text" name="title" placeholder="Article Title" defaultValue={title} required />
```

4 input 모두 적용 의무 (다른 form 일관성 정합).
