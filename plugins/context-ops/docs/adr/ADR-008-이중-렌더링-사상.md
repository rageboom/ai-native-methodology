# ADR-008: 이중 렌더링 사상 — 단일 진실 + AI 눈 + 사람 눈

> ⚠️ **Superseded by [ADR-011](ADR-011-json-only-ax-native.md)** (2026-06-01 / v12.0.0 MAJOR). 본 ADR 의 "두 청중(AI 눈 = `.json`/`.yaml` / 사람 눈 = `.md`/`.mermaid`) 동시 산출" 의무는 폐기됨 — 산출물 = **json 단독 SSOT**. 실측(2 distinct domain RealWorld·ecommerce): LLM·모든 검증 도구는 json 만 입력으로 읽고, `.md`/`.mermaid` twin 은 LLM 운영 컨텍스트에 0 기여 + 2차 렌더링 drift 표면만 추가. 본 ADR 의 "단일 진실(SSOT)" 원칙은 보존·강화(json 이 유일 진실)되나 "사람 눈 동시 산출" 의무는 무효. 본문은 역사 기록으로 보존. → DEC-2026-06-01-json-only-ax-native.

## 컨텍스트

본 방법론의 모든 산출물 은 **이중 형태로 동시 산출**된다:

| 영역         | AI 눈                    | 사람 눈                                |
| ------------ | ------------------------ | -------------------------------------- |
| Inventory    | inventory.json           | tree.md + stack-detection.md           |
| DB           | schema.json + schema.sql | erd.mermaid                            |
| Architecture | architecture.json        | architecture.mermaid + architecture.md |
| Domain       | domain.json              | domain.mermaid + domain.md             |
| API          | openapi.yaml             | api.md                                 |
| Rules        | rules.json               | rules.md                               |
| Antipatterns | antipatterns.json        | avoid-list.md                          |

그러나 **이 패턴이 우연이 아니라 사상**이라는 점이 명시되지 않았다. PoC #02 종결 후 사용자가 한 줄로 명시화:

> "AI에게도 눈을 사람에게도 눈을 달아주고 싶은거야"

본 결단은 ADR-001 (Schema-First) 의 *진짜 의미*가 이것이었음을 확정한다. Schema-First 는 "기계 검증" 만 강조하지만 실제로는 "기계 + 사람 모두를 위한 표준 형식"이었다. 그러나 ADR-001 은 사람 눈 (.md/.mermaid) 의 의무를 명시하지 않아 **갭이 발생**.

PoC #02 Phase 4.5 형식화 시범 (Sprint 1+1.5+2 누적) 에서:

- 이중 렌더링 정합도: 67% → 87.5% → 90.5%
- drift 누적: 4 → 15 → 34건
- 두 렌더링이 어긋날 때마다 = 단일 진실 손실 = 의사결정 분기

→ 본 사상을 명시화하지 않으면 v1.2.0 격상 시 묶음 L (Phase 4.5 정식 도입) 의 신규 산출물 5종 (state-machine / sequence / decision-table / invariants / property-test) 도 **임의로 한쪽만 만들어질 위험**.

## 결정

**"이중 렌더링 사상"** 을 본 방법론의 사상적 기반으로 공식 명시화한다.

### 정의

```yaml
이중 렌더링 사상 (Two-Eyes Principle):
  단일 진실 (Single Source of Truth):
    - 모든 산출물은 하나의 사실에서 출발
    - drift = 단일 진실 손실 = 의사결정 분기

  두 청중 (Two Audiences):
    AI 눈:
      형식: .json / .yaml / .schema.json / .ts
      용도: schema validation / 코드 자동 생성 / drift 검출 / CI 검증
    사람 눈:
      형식: .md / .mermaid
      용도: GitHub 자동 렌더 / 리뷰·학습·수정 / PR 토론

  의무:
    - 신규 산출물 추가 시 양쪽 렌더링 동시 의무
    - 한쪽만 있으면 ADR-008 위반 (ADR-002 산출물 정의 위반)
    - drift 자동 검증 (CI) 으로 일치 강제 (v1.2.0 묶음 N)
```

### 학문적 계보

#### Donald Knuth, "Literate Programming" (1984)

> "Instead of imagining that our main task is to instruct a computer what to do, let us concentrate rather on explaining to human beings what we want a computer to do."

본 사상 = Literate Programming 의 **AI 시대 재해석**:

- Knuth: 단일 파일에 코드 + 설명 (사람이 컴퓨터에게 / 사람에게 동시 설명)
- 본 방법론: 단일 진실에서 두 형식 동시 산출 (AI / 사람 동시 청중)

#### Spec-First / Schema-First 산업 표준

- Microsoft TypeSpec (2023~) — 단일 spec 에서 OpenAPI / Protobuf / JSON Schema 동시 생성
- Backstage Catalog (2020~) — YAML 단일 진실 → UI 자동 렌더
- OpenAPI 3.x — YAML/JSON 진실 → ReDoc/Swagger UI 사람 렌더

본 방법론은 산업 표준 패턴을 사상으로 격상한 것.

## 결과

### 긍정적 영향 (7건)

1. **PoC 산출물 갭 자동 식별** — 사상 위반 검출 명확화 (Sprint 2 90.5% 측정 가능)
2. **신규 산출물 추가 의무화** — Phase 4.5 5 산출물부터 양쪽 렌더링 강제
3. **Drift 자동 검증 근거** — v1.2.0 묶음 N (CI 도구) 의 사상적 정당화
4. **AI 코드 생성 신뢰도 향상** — AI 눈 (JSON) 으로 코드젠 입력 / 사람 눈 (Mermaid) 으로 사람 검토
5. **사용자 onboarding 가속** — 사람 눈 (.md/.mermaid) 으로 빠른 학습
6. **방법론 본체 자체의 정합 강제** — 묶음 M 갭 7건 식별 근거 (DEC-2026-04-30)
7. **학문적 계보 명시** — Literate Programming AI 시대 재해석으로 정통성 확보

### 부정적 영향 / 위험 (3건)

1. **산출 비용 ~2배** — 모든 산출물에 양쪽 렌더링 강제 (단일 .md 보다 많음)
   - **완화**: 단일 진실 (.json) 에서 사람 눈 (.mermaid) 자동 생성 도구 도입 (v1.2.0 묶음 N CI 통합 시 검토)
2. **Drift 위험** — 양쪽이 갈라질 가능성
   - **완화**: CI 검증 의무 (묶음 N) + cross-validation 패턴 정착 (Sprint 1.5+2 정합)
3. **신규 산출물 기준 모호** — 어디까지 양쪽 의무인지 (예: 작업 노트도 양쪽? )
   - **완화**: ADR-002 산출물 + Phase 4.5 5 산출물 + 메타 명시 영역만 의무. plan/research/PROGRESS 등 작업 문서는 사람 눈만.

## 대안 (검토 후 거부)

### 대안 1: 단일 .md 만 (사람 눈만)

- **장점**: 산출 비용 1배 / 학습 곡선 낮음
- **거부 이유**: AI 코드 생성 / schema validation 불가 → 본 방법론의 "AI-Native" 정체성 손실

### 대안 2: 단일 .json 만 (AI 눈만)

- **장점**: 단일 진실 절대 보장 / drift 0
- **거부 이유**: 사람 검토/PR 토론 어려움 → 사용자 onboarding 차단 / GitHub 자동 렌더 불가

### 대안 3: 양쪽이지만 별도 진실 (현재 산업 anti-pattern)

- **장점**: 자유도 높음
- **거부 이유**: 두 진실 = drift 발생 → Sprint 2 에서 34건 발견 (정합 90.5%)

→ **이중 렌더링 (단일 진실 + 두 형식) 만이 AI-Native 방법론의 정합 옵션**.

## 신규 산출물 추가 시 체크리스트

본 ADR 채택 후 모든 신규 산출물 추가 시:

```
□ AI 눈 형식 결정 (.json / .yaml / .schema.json / .ts)
□ 사람 눈 형식 결정 (.md / .mermaid)
□ 단일 진실 정의 (어느 쪽이 SSOT 인가)
□ 자동 변환 가능성 검토 (단일 진실 → 다른 렌더링)
□ Drift 검증 전략 (수동 / CI)
□ 신뢰도 등급 (ADR-009 — 묶음 M P2 신규 예정)
```

## ADR-001 과의 관계

ADR-001 (Schema-First) 은 본 ADR 에 **포섭되지만 supersede 되지 않음**:

- ADR-001: "기계 검증을 위한 schema 우선" (기존)
- ADR-008: "AI + 사람 모두를 위한 이중 렌더링 우선" (확장)

ADR-001 의 Schema-First 는 본 사상의 *AI 눈 부분*에 해당. ADR-001 자체는 그대로 유효.
