- id: `internal.fe.security.jwt-localstorage` (fully qualified slug / `--rewrite-rule-ids` default ON 실측 정합)
- severity: `HIGH` (canonical)
- languages: `[js, ts]`
- pattern-either 4 분기 (점진 발견 / Phase A → B → C 보강):
  1. `localStorage.setItem("$KEY", $VAL)` + metavariable-regex
  2. `localStorage.setItem($KEY, $VAL)` + metavariable-regex (identifier branch)
  3. `window.localStorage.setItem("$KEY", $VAL)` + metavariable-regex (window. prefix)
  4. `window.localStorage.setItem($KEY, $VAL)` + metavariable-regex (window. + identifier)
- metavariable-regex: `(?i)(token|jwt|auth|access|bearer|session)` (Python re inline flag / left-anchored)
- metadata: cwe CWE-922 / owasp [A02:2021, A07:2021] / references / category security

`tools/static-runner/rules/jwt-localstorage.ts` (Semgrep convention — test fixture는 `<rule>.<ext>` / `.test.<ext>` ❌ 실측 정정):

- 7 positive cases (ruleid: marker)
- 3 negative cases (ok: marker — preferences / theme / sessionStorage)
- 검증: `semgrep --test --config jwt-localstorage.yml jwt-localstorage.ts` → **1/1 pass ✅** (Windows 한국어 환경 PYTHONUTF8=1 의무 — yml 한글 message cp949 decode bug 회피)

## 8. Lessons Learned

1. **Custom rule pattern 점진 발견 패턴** — 단순 pattern → 분기 (string vs identifier) → 분기 추가 (window. prefix) = 4 분기 의무. 실 코드 진짜 매칭 시 pattern 보강 의무 (일반화 가능).
2. **Semgrep `--rewrite-rule-ids` default ON 실측** — Official research Q4 carry 해소. fully qualified slug 권고 정합 입증.
3. **sarif-to-finding 어댑터 severity 변환 (HIGH → medium) 검토 필요** — 본 carry scope 외부 / 별도 carry 등재.
4. **Sprint 4 long-tail carry (사내 custom rule "별도") 같은 날 종결** — implicit 목표 trigger 시 즉시 종결 가능 패턴.
5. **같은 날 v1.4.0 + v1.4.1 + v1.4.2 = Auto Mode + 4원칙 + 일괄 승인 패턴 = 빠른 carry resolve cadence 입증** .
