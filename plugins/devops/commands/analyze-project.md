# 프로젝트 분석

특정 프로젝트의 구조, 의존성, 설정을 분석합니다.

## 사용자 입력
- $ARGUMENTS: 프로젝트명 (예: "eam", "front", "observer")

## 지침

### 분석 항목

1. **프로젝트 구조**
   - 모듈/패키지 구조 파악
   - 주요 디렉토리 설명
   - 엔트리 포인트 확인

2. **의존성 분석**
   - 주요 라이브러리 버전
   - 내부 모듈 의존 관계
   - 외부 서비스 연동

3. **설정 파일 검토**
   - 프로파일별 설정
   - 환경 변수 목록
   - 보안 관련 설정

4. **API 엔드포인트** (Backend)
   - Controller 목록
   - API 경로 패턴
   - Swagger 문서 위치

5. **컴포넌트 구조** (Frontend)
   - 페이지 라우팅
   - 공유 컴포넌트
   - 상태 관리

6. **빌드 설정**
   - 빌드 스크립트
   - 환경별 빌드 명령
   - 아티팩트 위치

### 출력 형식
- 마크다운 테이블로 정리
- 중요 파일 경로 명시
- 권장 사항 포함
