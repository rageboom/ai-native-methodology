1. **★ Custom rule pattern 점진 발견 패턴** — Phase A 단순 pattern → Phase B 분기 (string vs identifier) → Phase C 분기 추가 (window. prefix) = ★ 4 분기 의무. ★ 실 코드 첫 진짜 매칭 시 pattern 보강 의무 (★ 일반화 가능 패턴).
2. **★ Semgrep `--rewrite-rule-ids` default ON 실측** — Official research Q4 carry 해소. SARIF ruleId = `<cwd-relative-path>.<rule-id>` (★ `--config` 멀티 지정 시 자동 prefix). fully qualified slug 권고가 정합 입증.
3. **★ sarif-to-finding 어댑터 severity 변환 (HIGH → medium) 검토 필요 carry** — 본 carry scope 외부 / ADR-010 §2.3 정합 검토 별도 (★ v1.4.3 후보).
4. **★ Sprint 4 README 의 "사내 custom rule (별도)" 1년 carry 같은 날 종결** — long-tail carry 가 implicit 목표 trigger 시 즉시 종결 가능 패턴.
