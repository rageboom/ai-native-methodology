## 컨텍스트

DEC-round-trip-스코프-아웃 결단 후 사용자 의도:

> "기존 시스템에 버그나 위험한 것들, 혹은 부조리가 있다면 이것도 한켠에 다 기록하면 좋을것 같기도해."

신규 시스템 자동 생성은 안 하지만, **신규 시스템 만드는 사람이 회피해야 할 패턴 가이드** 가 산출물로 도움이 되어야 함.

현재 (PoC #01 + #02 종결 시점):

- antipatterns.json — 36 AP (critical 5 / high 5 / medium 18 / low 8)
- avoid-list.md — priority order + composite view 4건
- findings/poc-findings.md — 76 finding

→ "AP 가 무엇인지" 는 명시. "신규 시스템에서 어떻게 회피하는지" 는 부분만.

## 결정

**강화점 α + β 채택**:

### α — `antipatterns.json` schema 에 `migration_advice` 필드 추가

각 AP 에 다음 항목 추가:

```json
{
	"id": "AP-API-001",
	"severity": "critical",
	"name": "spec/runtime status code drift",
	"description": "...",
	"migration_advice": {
		"do_not": [
			"status code 200 OK on idempotent state-changing operations without spec compliance"
		],
		"do": [
			"RFC 7231 §4.2.2 + RFC 9110 §15.3 정합",
			"Profile pattern: if (already) return;"
		],
		"verify": ["openapi spec ↔ runtime drift 자동 검증 (CI)"],
		"applicable_to_new_system": true,
		"language_neutral": true
	}
}
```

**필드 설명**:

- `do_not` — 신규 시스템에서 절대 하지 말 것 (구체 행동)
- `do` — 대신 할 것 (구체 가이드)
- `verify` — 자동 검증 방법 (CI / 도구)
- `applicable_to_new_system` — 신규 시스템 영향 여부 (legacy 한정 패턴 분리)
- `language_neutral` — 언어/스택 무관 여부

### β — `migration-cautions.md` 신규 산출물 (Phase 6 의무)

사람-읽기 전용 단일 문서. critical / high AP 만 추려서 "신규 시스템 구축 시 절대 하지 말 것".

**구조**:

```markdown
# Migration Cautions — 신규 시스템 구축 시 회피 가이드

## Critical (반드시 회피 — 5건)

### AP-API-001 — spec/runtime status code drift

- 절대 하지 말 것: ...
- 대신: ...
- 자동 검증: ...

## High (강력 권고 회피 — 5건)

...

## 영역별 composite view (5건)

- §A — API 계약 결함
- §B — Hexagonal port-adapter 경계
- §C — JWT/Auth 보안
- §D — 데이터 무결성
- §E — Crypto 자체 구현
```

→ `avoid-list.md` 와 차이:

- `avoid-list.md` = 분석 대상 시스템에 대한 발견 목록 (현재 주체)
- `migration-cautions.md` = 신규 시스템 구축자에게 주는 가이드 (미래 주체)

## 근거

### 1. 사용자 직관 정합

"한켠에 다 기록" → 별도 단일 문서 (β) + 구조화 데이터 (α) 둘 다 필요.

### 2. 이중 렌더링 사상 정합

- α (JSON 필드) = AI 눈
- β (MD 문서) = 사람 눈
- 단일 진실 (antipatterns.json) → 두 렌더링 (α 자동 / β 사람 작성 또는 자동 추출)

### 3. 신규 시스템 입력 자료로의 가치 강화

DEC-round-trip-스코프-아웃 §2 "산출물 5종 이식성" 정합:

- BR / 도메인 / API / Schema = 입력 자료 (positive — 만들어야 할 것)
- antipatterns + migration-cautions = 회피 가이드 (negative — 하지 말 것)

→ positive + negative 둘 다 제공 시 신규 시스템 입력 자료로 완성도 높음.
