## Lessons Learned 등재 ( session 25차 / ADR-CHAIN-011 §9)

- **LL-i-52** — alias 다중첩 폐기 시 "순수 rename alias" vs "의미 구분 별개 키" 분리 식별 의무 (`rules_auto_extracted_reference` = object/provenance ≠ BR-list alias → ① scope 외). 일괄 폐기 ❌ / 의미 보존 우선. + scope-completion 투명 명시 paradigm (`br_summary` = 사용자 열거 외였으나 동일 class → 의도 정합 위해 포함하되 DEC 명시 / silent 확장 ❌).
- **LL-i-53** — breaking migration 이 latent consumer 결함(poc-04 br-cross-consistency 비가시 / extractRules rules_manual_authored 미처리)을 동시 수정 = breaking change 의 숨은 가치. 단 "회귀 vs 개선" 판정 = 코드 착수 전 dry-run 실측 의무 (Senior STOP-1 / ⑤ both=0 실측과 동형 paradigm). OpenAPI 3.1 latent-bug-fix precedent 정합.
