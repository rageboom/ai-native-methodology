## 컨텍스트

"PoC #02 산출물로 바로 백엔드 코드 만들 수 있어?" 질문 제기.

본 방법론 v1.1.2 까지 검증 범위:
- ✅ 코드 → 7대 산출물 (PoC #01 + #02)
- ❌ 7대 산출물 → 코드 (PoC 미실시)

## 결정

**round-trip 미검증을 명시 인지**. 공식 미해결 영역으로 등록. 답변 정확화: "스켈레톤 즉시 / 동작 백엔드 1~2주 추가 작업 / 정식 검증 미실시".

## 근거

방법론이 **분석(reverse engineering) 전용** 으로 설계됨. 산출물은 "AI 재구현 입력" 으로 만들어졌으나 round-trip 자체는 검증된 적 없음. PoC #01/#02 모두 분석 방향만.

## 영향

### 즉시 가능한 것 (도구만)
- schema.sql → DB 즉시 재현
- openapi.yaml → openapi-generator 컨트롤러/DTO/SDK 자동 생성

### AI 추가 시 (1세션, ~70~80%)
- Entity / Repository / Controller 시그니처 / DTO 변환

### 추가 작업 필요 (방법론 외부)
- Service 본문 (rules.md 추론), Auth/Tx, 외부 시스템 연동, 설정, 테스트

### 시간 추정 (1인 중간 규모)
- 분석 → 1세션
- 스켈레톤 → 30분
- AI 본문 채우기 → 2~4시간
- 비즈니스 로직 → 1~2일 (수동 검증 필수)
- Auth/Tx/연동/설정/테스트 → 2~5일
- **총 약 1~2주**
