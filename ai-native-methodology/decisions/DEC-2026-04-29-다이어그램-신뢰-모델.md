## 컨텍스트

Sprint 1 산출물 (FSM/Sequence/Decision Table/ADT) 검토 중 사용자 질문:
> "다이어그램이 틀릴수도 있나?"

→ 다이어그램 신뢰도가 절대값 100% 가 아닌 **검증 단계별 가변** 임을 명시화 필요.

## 결정

**Phase 4.5 형식 명세의 "단계적 신뢰 모델" 공식 채택**.

다이어그램은 "정답" 이 아니라 **"단계적으로 신뢰 쌓는 자산"**. 사용 시 신뢰도 명시 + 용도별 가이드 적용.

### 신뢰도 정량 모델 (정직 보강 — DEC-static-tool-실행-의무화 반영)

| 검증 단계 | 신뢰도 | 설명 | 도구 종류 |
|---|---|---|---|
| 1차 작성 (사람·AI 단독) | 60~70% | 초기 추출 | - |
| + Cross-validation AI sub-agent (F-015) | 75~85% | 독립 sub-agent 비교 | AI sub-agent (Claude) |
| + ~~Static Analyzer persona sub-agent~~ | 80~87% | Sub-agent 시뮬레이션 (진짜 도구 ❌) | ★ -5%p 패널티 |
| + 진짜 정적 도구 실행 | **85~92%** | Daikon/Semgrep/PMD/SpotBugs/CodeQL 실제 실행 | ★ 진짜 도구 |
| + Property test 실행 | 90~95% | jqwik/Vitest 실제 실행 | - |
| + 사람 도메인 전문가 리뷰 | 95%+ | critical 영역 | - |
| + L4 (TLA+/Coq) | 이론적 100% | critical only | - |

**★ 도구 종류 구분 의무**:
- AI sub-agent (Claude/Gemini 등) — "사람-like AI" 또는 "도구 persona 시뮬레이션"
- 진짜 외부 도구 (Daikon/Semgrep/PMD/SpotBugs/CodeQL/SonarQube) — 실제 실행 결과
- **두 종류 절대 동일 신뢰도로 표기 ❌** (DEC-static-tool-실행-의무화)

## 근거

### 6 가지 오류 원인 (사실 인정)
1. 추출 오류 (사람·AI 한계)
2. 모델 단순화 손실 (Mermaid 표현력 한계)
3. Source of Truth 함정 (코드 진실 가정 위험)
4. 형식 표기 자체 한계 (nested/parallel state)
5. Drift (시간 경과 동기화 부재)
6. 사람 모델 편향 (추상화 임의적)

### 본 Sprint 1 다이어그램의 알려진 한계 (자가 평가)
- `User-Account.mermaid`: nested state 단순화 / F-058 노트만 / JPA tx boundary 미표현
- `UC-USER-SIGNUP.mermaid`: race interleaving 시점 한계 / Spring @Transactional 미표현
- `BR-USER-EMAIL-UNIQUE-001.md`: 안전 등급 주관 / race window 빈도 미정량 / F-054 caveat 미반영
- `User.ts`: createdAt immutable 단일 표현 / NoSuchEx vs IAE 구분 부족

### 검증 방법 5종
1. Cross-validation (F-015) — sub-agent 독립 추출 비교
2. Property test 실행 — FSM transition / invariant 자동 검증
3. 컴파일 검증 (Direction D) — self-reference 함정 주의
4. 사람 리뷰 — 도메인 전문가 통찰
5. Static analysis tool 교차 — Daikon, JADET, Semgrep 등

## 영향

### Phase 4.5 산출물 메타데이터 의무화
모든 형식 명세 산출물에 신뢰도 메타 명시:
```yaml
trust_level:
  current: 70%             # 현재 단계
  validation_history:
    - {step: "1차 작성", at: "2026-04-29", actor: "AI"}
    - {step: "cross-validation", at: "2026-04-29", actor: "sub-agent senior"}
  next_validation: "property test (Sprint 4)"
known_limitations:
  - "nested state 단순화"
  - "race interleaving 정량 부재"
```

### 사용 가이드 (각 산출물에 명시 의무)
- 신뢰해도 되는 용도 (온보딩, PR 리뷰, drift 발견, 코드젠 입력)
- 신뢰하면 안 되는 용도 (critical 시스템 단독 근거, 컴파일 없이 prod, race 안전 단독 증명)
