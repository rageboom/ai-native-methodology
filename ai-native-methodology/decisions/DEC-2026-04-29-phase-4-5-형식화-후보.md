## 컨텍스트

"서비스 로직을 컴퓨터 사이언스 관점에서 표기 가능하지 않나?" 질문 제기.

현재 `rules.json` Given/When/Then 은 **자연어 문자열** (L0). AI 코드 생성 정확도 천장 ~70%.

```json
"given": "시스템에 email='X' 인 User 가 이미 존재할 때",
"when": "동일 email 로 signup 요청이 들어오면",
"then": "IllegalArgumentException 발생 + DB unique 제약이 race-safe 차단"
```

## 결정

**Phase 4.5 (형식화) 신규 phase 후보 등록**. v1.2.0 또는 v1.3.0 격상 시 ADR-008 후보. 즉시 도입 ❌, 검토중.

## 근거

### 형식화 5단계
| Level | 표기법 | AI 코드 생성 정확도 | 추가 비용 |
|---|---|---|---|
| L0 (현재) | 자연어 G/W/T | 60~70% | 0 |
| **L1** | + Decision Table + FSM | 80~85% | +20% |
| **L2** | + ADT/Refinement Type + Invariant | 85~92% | +40% |
| L3 | + 구조화 의사코드 (Hoare triple) | 90~95% | +60% |
| L4 | + TLA+/Alloy/Z | 95%+ 검증 가능 | +200% |

→ Sweet spot: **L1~L2 default / L3 critical 경로 only / L4 분산·동시성 한정**.

### 영역별 적합 표기법
| 영역 | 권고 |
|---|---|
| Validation | Refinement Type / Decision Table |
| State transition | FSM (Mermaid stateDiagram) |
| Authorization | ABAC predicate |
| Workflow | Sequence Diagram + 의사코드 |
| Idempotency | Algebraic property (`f ∘ f = f`) |
| Race condition | Linearizability spec / TLA+ |
