## 컨텍스트

DEC-2026-04-29-시퀀스-C-A-B-확정 §컨텍스트 4 항목에 "**방법론 본체 자체의 이중 렌더링 갭 7건 (Priority 1~5)**" 발견 명시.

그러나:
- 갭 7건의 **구체 리스트 미문서화** — 발견만 되고 식별 작업 미수행
- v1.2.0 격상 (시퀀스 A) 시 묶음 M 으로 통합 처리 예정인데 **무엇을 채울지 미정**
- "별도 작업 / 병행 가능" 으로 표기되어 우선순위 불명확

→ 본 DEC 로 **갭 7건 공식 식별 + Priority 부여 + 처리 계획 확정**.

## 근거

### 식별 방법
방법론 본체 전수 조사 — 다음 디렉토리/파일 점검:
- `methodology-spec/` (deliverables 7 + workflow 8 + finding-system + 용어집/규약)
- `schemas/` (8 schema + meta-confidence + README)
- `templates/` (14 template — .md / .mermaid / .json / .yaml)
- `docs/adr/` (ADR-001~006)

각 영역의 **AI 눈 (`.json`/`.yaml`/`.schema.json`) ↔ 사람 눈 (`.md`/`.mermaid`)** 짝 매핑 점검.

### 갭별 상세

#### #1 finding-system.schema.json 부재 (P1)
- **현황**: `methodology-spec/finding-system.md` 만 존재 (DRAFT / 자산화 대기)
- **누락 영향**:
  - finding 등록 시 schema validation 불가 (수동 YAML 작성)
  - severity / disposition / spec_gap 필드 enum 강제 불가
  - finding 누적 통계 자동 집계 불가
- **memory 정합**: `project_finding_system.md` 명시 — "draft / 자산화 대기"
- **차단 영향**: v1.2.0 격상 시 finding-system 정식화 ↔ schema 부재로 격상 자체가 미완

#### #2 ADR-008 (이중 렌더링 사상) 정식 본문 미작성 (P1)
- **현황**: DEC-2026-04-29-이중-렌더링-사상-명시화 만 등록
- **DEC §"ADR 격상 후보"** 명시: "ADR-008 신규 또는 ADR-001 supersede"
- **누락 영향**:
  - v1.2.0 묶음 L (Phase 4.5 정식 도입) 격상 시 사상적 기반 부재
  - 신규 산출물 추가 시 이중 렌더링 의무 근거 부재 (DEC 만으로 약함)
  - Donald Knuth Literate Programming 계보 학문적 근거 명시 필요

#### #3 api.template.md 부재 (P2)
- **현황**: `templates/api.template.yaml` 만 존재
- **PoC 사례**: PoC #01 / #02 모두 `api.md` 직접 작성 (template 없이)
- **누락 영향**: 사람이 OpenAPI YAML 직접 읽기 어려움 — 요약 .md 필수

#### #4 phase-flow.mermaid 부재 (P2)
- **현황**: `phase-N-*.md` 8개 파일 (phase-0 / 1 / 2 / 3 / 4 / 5-1 / 5-2 / 6) 텍스트만
- **누락 영향**:
  - 신규 사용자 onboarding 시 전체 흐름 파악 어려움
  - Phase 4.5 도입 위치 시각화 불가 (현재 phase-4 와 phase-5-1 사이 끼어드는 흐름)
  - 산출물 의존 그래프 (어느 phase 산출물이 어느 phase 입력인지) 시각화 부재

#### #5 ADR-009 (다이어그램 신뢰 모델) 미작성 (P2)
- **현황**: DEC-2026-04-29-다이어그램-신뢰-모델 + memory `feedback_diagram_trust_model.md` 만
- **누락 영향**:
  - Phase 4.5 형식 명세 신뢰도 60~95% 가변 모델 정합 불가
  - 검증 5단계별 신뢰도 변화 정식 등록 부재
  - 사용자/하네스가 "이 다이어그램의 신뢰도 = N%" 판단 근거 부재

#### #6 db-schema.template.md 부재 (P3)
- **현황**: `templates/erd.template.mermaid` 만 존재
- **PoC 사례**: PoC #02 `정합성-검증-보고서.md` 형태로 직접 작성
- **누락 영향**: PoC 마다 사람 눈 요약 형식 발명 — 표준 부재

#### #7 meta-confidence template 부재 (P3)
- **현황**: `schemas/meta-confidence.schema.json` 만 존재
- **PoC 사례**: 각 phase `_manifest.yml` 로 자체 발명
- **누락 영향**: 산출물별 메타 작성 규약 명시 부재 → 일관성 저해

### 갭 7건의 공통 메커니즘
모든 갭은 **DEC-2026-04-29-이중-렌더링-사상-명시화** 의 사상적 기반 위반:
- "단일 진실 (SSOT) + 두 렌더링 (AI 눈 / 사람 눈)" 사상에 본체 자체가 미적합
- → 본 사상이 PoC 산출물에는 강제되나 본체에는 미강제 → 본체가 *방법론을 따라가지 못하는 상태*
