## 5. Lessons Learned 신규 ( session 11차 / ADR-CHAIN-011 §9 정합)

-     **LL-i-33** ( "trigger-condition-action 4축 → given-when-then 3축 형식 sliding paradigm = Cucumber/Fowler/ECA 3중 외부 권위 정합 / action = GWT step 분리 ❌ + metadata 보존"):

  - **Why**: Cucumber 공식 "When = single action only" anti-pattern + Fowler "when=behavior/trigger" + ECA "Condition=precondition=Given" 3중 정합. action = "시스템 동작 (e.g. @IsNotEmpty 검증)" = Gherkin GWT 외부 보존 자산 (rejection_method + verification_location) 영역.
  - **How to apply**: TCA paradigm BR → GWT 합성 시 = "trigger→When / condition→Given / expected_result→Then 3축 형식 sliding + action = metadata 보존" 의무. LLM 부재 시 = 형식 sliding only 의무 ( semantic 합성 ❌ / NL = TODO marker carry).

-     **LL-i-34** ( "Layer 1 keyword threshold 자체 제거 paradigm = 'non-empty + overlap > 0' sanity check only / Layer 2 LLM mandatory Phase C 의무"):

  - **Why**: session 9차 SPIKE v2 결정적 사실 (Jaccard short-text + 한국어 교착어 형태소 부재 = ≥ 0.85 수학적 도달 불가) + Agent 3 STOP-3 (magic number 자산화 회피) + Agent 1 외부 권위 (MDPI 2025 "no single generalizable cut-off") 3중 정합.
  - **How to apply**: Layer 1 = structural sanity check 격하 ( 키워드 비교 자체 = 진위 ❌) / Layer 2 LLM = mandatory 격상 ( semantic 정합 검증 영역) / threshold magic number 자산화 회피 의무. 외부 인용 시 "Layer 1 = sanity / Layer 2 = semantic" paradigm 명시 의무.

- **LL-i-35** ( "industry-first 자격 scope 정정 / 'dual representation' = Cucumber Rule 키워드 (2018) 정합 / 'cross-consistency validator (NL ↔ GWT 자동 검증)' + '4축 → GWT deterministic 합성 도구' = industry-first 자격"):
- **Why**: Agent 1 결정적 사실 — Cucumber Rule 키워드 (2018) + Gáspár Nagy "story-rule-scenario" 가 이미 dual representation paradigm 정의. 본 방법론 industry-first 자격 ≠ "dual representation 자체". Agent 2 결정적 사실 — Spec Kit (90K star) / AutoUAT / CodeConcise / Spec Kit / Drools / IBM ODM 모두 "NL ↔ GWT cross-consistency validator" + "4축 → GWT deterministic 합성 도구" = 공개 자료 0건.
- **How to apply**: 외부 인용 시 = industry-first 자격 = "cross-consistency validator + 4축 → GWT deterministic 합성" scope 한정 의무. "dual representation" 자체는 Cucumber Rule 정합 명시 의무.
