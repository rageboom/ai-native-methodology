# F-FE-003 — JWT localStorage XSS exposure (★ PoC #01/02 isomorphic 변형)

> ★ Stage 4 mini-PoC finding
> 일자: 2026-05-02
> 분류: medium / candidate (★ §8.1 정합 / 단일 PoC 발견)

## Where

`INPUT/src/shared/api/auth-storage.ts` (line 12, 20, 28)

## What

```ts
window.localStorage.setItem(TOKEN_STORAGE_KEY, token);   // line 20
return window.localStorage.getItem(TOKEN_STORAGE_KEY);    // line 12
window.localStorage.removeItem(TOKEN_STORAGE_KEY);        // line 28
```

JWT 토큰을 `localStorage` 에 저장.

## Why

OWASP A02:2021 — Cryptographic Failures (또는 A07 — Identification and Authentication Failures).

XSS 공격 발생 시 (★ 본 fork 의 다른 XSS pattern 0건 — F-FE-002 의 `html-has-lang` 외) `document.cookie` 와 달리 localStorage 는 ★ JavaScript 직접 read 가능 → 공격자 token 탈취 위험.

## Severity

medium (★ 본 fork 의 XSS surface 작음 / 단 JWT 탈취 시 모든 endpoint 인증 우회 가능).

## Recommendation (BE 협조 필요)

- ✅ 권장 1: httpOnly cookie + CSRF token 패턴 (BE 의 Cookie 전송 + SameSite=Lax/Strict 설정)
- ✅ 권장 2: short-lived access token (5min) + refresh token (httpOnly cookie)
- 단기 완화: ★ 현 fork 가 RealWorld 표준 / 패턴 유지 시 ★ ★ 강한 CSP (script-src) 적용 권장
