# F-FE-001 — Article delete 후 redirect 부재

> ★ Stage 4 mini-PoC finding
> 일자: 2026-05-02
> 분류: medium / candidate (★ Stage 5 promote 후보 — §8.1 정합 / 1 PoC 단독)

## Where

`INPUT/src/pages/article/article.ui.tsx` § DeleteArticleButton (line 212-232)

## What

```tsx
const handleDeleteArticle = () => {
  deleteArticleFetcher.submit(null, { action: articlePaths.getDeletePath(slug), method: 'delete' });
};
```

## Why

server delete 성공 후 redirect 명시 부재 → 클라이언트가 동일 article slug URL 잔존 → 다음 새로고침 시 404 또는 stale state.

## Severity

medium — UX 문제 (data loss 아님 / silent failure 가능).

## Recommendation

action 결과를 react-router redirect 로 처리 — `actions/article-delete.action.ts` 에 `return redirect('/')` 추가.
