## 2. ★ ★ ★ 본체 도구 격상 (★ static-runner 0.1.0 → 0.1.1)

본 carry 의 첫 진짜 실행 = ★ 본체 도구 bug 2건 발견. ★ ★ ★ no-simulation 정책 적용 = 도구 자체 가치 검증 효과 입증 ★.

### 2.1 Bug 1 — `result_hash: null` (★ ★ ★ no-simulation 핵심 필드 위배)

- 위치: `tools/static-runner/src/runner.js:71`
- 원인: `require('node:fs').readFileSync(sarifPath)` ESM 환경 throw → catch silently swallow → `resultHash = null`
- 영향: ★ ★ ★ no-simulation 정책 핵심 필드 (위조 차단) 효과 깨짐
- Fix: `import { readFileSync, ... } from 'node:fs'` 추가 + `h.update(readFileSync(sarifPath))`

### 2.2 Bug 2 — `source_commit_sha: "unknown"` (★ ratchet reproducibility 깨짐)

- 위치: `tools/static-runner/src/cli.js`
- 원인: `writeBaseline(opts.writeBaseline, allFindings)` 호출 시 `sourceCommitSha` 인자 누락 → `'unknown'` 기본값
- 영향: baseline 의 reproducibility 부재 (어느 commit baseline 인지 추적 불가)
- Fix: `detectGitSha(targetDir)` 함수 신설 + `git rev-parse HEAD --target-dir-cwd` 추출 후 전달

### 2.3 검증

- 9/9 unit test pass ✅ (기존 test 깨뜨리지 않음)
- evidence.json 재실행 시 `result_hash` SHA256 정상값 + `source_commit_sha` 12-char prefix display ✅

1. **★ release note 의 carry 환경 가정 검증 누락 패턴** — pip/pipx/uv 등 다른 진짜 도구 채널 옵션 검증 의무. carry 명시 시 "환경 가정" 컬럼 별도 + "다른 채널 가능성 검증 여부" 명시 의무 (★ 신규 패턴 / memory `feedback_carry_environment_assumption.md` 신규).

2. **★ ★ ★ no-simulation 정책 첫 적용 시 도구 자체 가치 검증** — 진짜 도구 첫 실행 = 본체 도구 bug 발견 가능. ★ no-simulation 정책 = 도구 자체의 가치 검증 효과까지 동반. ★ feedback_no_simulation_realized.md 패턴 강화.

3. **★ Senior research 추정 over-estimate 입증** — RealWorld FSD 코드 품질 우수 (66 files / 76 rules / 0 findings). 작업 시간 4~8h → ★ 1.5h 단축.

4. **★ Silver bullet 부재 입증** — Semgrep p/owasp-top-ten 만으로 사내 핵심 AP 매칭 부족. 사내 적용 시 custom rule 작성 의무.
