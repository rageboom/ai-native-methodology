### 2.2 ★ cross-check 권고 3건 반영 사항 (의무)

#### 2.2.1 DTCG 정확한 인용 (#3)

- spec 명칭 = **"Design Tokens Format Module 2025.10 (Final Community Group Report)"**
- ★ **W3C Standard ❌** / Community Group Report — ADR / deliverable 인용 시 명시 의무.
- spec URL 고정 = `https://www.designtokens.org/TR/2025.10/format/`.
- 산출물 schema (ui-spec.schema.json design-tokens) 의 `$schema` 또는 `x-spec-source` 필드에 정확한 URL 저장 권장.
- 주요 필드 = `$value` (required) / `$type` (optional) / `$description` (optional).

#### 2.2.2 WCAG 2.2 AA ratchet path (#9)

- baseline = **WCAG 2.1 AA**.
- ratchet 격상 = **WCAG 2.2 AA** (REC 2023-10 / ISO/IEC 40500:2025 / W3C 사용 권고 중).
- ADR-010 (baseline+ratchet) 패턴 정합 = baseline 미달 시 fail / ratchet 격상 시 권장.
- axe-core config 의 `wcag_level` enum = `["2.1-AA", "2.2-AA"]` 양쪽 채택 의무.

#### 2.2.3 ICU MF2 채택 단계 명시 (#11)

- spec 자체 = **Stable** (CLDR LDML 47 부터 stable part of Unicode LDML).
- runtime ICU4J/ICU4C = **Technical Preview** (일부 default function / `u:` namespace 는 Draft 잔존).
- production 채택 위험도 carry-over 명시 (Stage 3-2 i18n deliverable 신설 시 본 ADR 인용 의무).
- 단기 폴백 = **ICU MF1 ko-KR 병기 의무** (Stage 4 mini-PoC 부터 적용).

## 4. 결과 (Consequences)

### 4.1 좋은 점

- **이식성 ★★★** — 12 매개체 spec 자체가 산업 표준 = 언어/환경 100% 무관 (CLAUDE.md ★★★ 가치 명세 정합).
- **★ AI 학습 코퍼스 의존 회피** — 1차 사료 spec URL 고정 (cross-check 1차 사료 의무).
- **사용자 7 요구사항 1번 직접 해소** — "산출물 → 마이그+테스트 기반". 12 매개체가 모두 자동 도출 가능 (Storybook CSF + Playwright + axe-core + MSW handler).
- **WCAG ratchet path 명시** — ADR-010 (baseline+ratchet) 패턴 재사용. 2.2 격상 시 코드 변경 0 (config 만).

### 4.2 나쁜 점

- 12 매개체 모두 입증은 Stage 4 mini-PoC + Stage 5 본격 PoC 까지 carry-over (Stage 3-1 = sub-agent 1차 사료 검증만).
- ICU MF2 runtime preview = production 위험. Stage 3-2 i18n deliverable 신설 시 위험 명시 의무.
- MSW + OpenAPI **3.1** specific 호환은 Stage 4 mini-PoC 에서 README 추가 검증 carry (cross-check 시 unverified).
- Storybook CSF v3 = 산업 표준이나 v3 syntax 강제 시 v2 legacy 호환 비용 (Tier 2/3 영향).

### 4.3 무시한 다른 옵션

- **매개체 6~7 경량 채택 (CEM / SCXML / DTCG / MSW+OAS / axe-core / Playwright / Pact)** — 거부. Storybook CSF + Storybook static 이 사람 눈 직접 렌더 = ADR-FE-002 정합 핵심. WCAG / WAI-ARIA / ICU 도 비기능 deliverable 의 진실 (Stage 3-2).
- **매개체 16+ 확장 채택** (Lighthouse / WebPageTest / Vitest / Cypress 추가) — 거부. 운영 NFR / E2E 다중화 = ADR-001 §명시적 제외 영역. Stage 5+ 검토.
- **DTCG W3C Standard 격상 대기 후 채택** — 거부. Community Group Report 도 산업 채택 ✅ (Tailwind / Material UI / Style Dictionary 모두 정합). W3C Standard 격상 대기 시 v1.5+ 지연.
